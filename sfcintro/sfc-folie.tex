\Pfoilhead{Sequential Function Charts (SFC)}

\begin{itemize}
\item SFC's: eine von mehreren standardisierten Beschreibungssprachen für
  Mikrokontroller-Programme
\item IEEE-Standard
\item Sprachen werden zusammen verwendet, um Mikrokontroller zu programmieren
  (d.h. eine Funktion eines Programmes in Sprache A kann in Sprache B
  geschrieben werden)
\item Assembler-artig, Pascal-artig, usw.
\item von vielen großen Firmen benutzt
\item SFC's: graphisch
\item Petrinetz-artig
\end{itemize}

\Pfoilhead{SFC's (3)}

\begin{itemize}
\item Steps = Knoten, assoziiert dazu sind Aktionen
  (z.B. \texttt{x\,:=\,false})
\item Transitionen mit Guards zwischen Steps
\end{itemize}

\bigskip

\begin{center}
  \textbf{\large Arbeitszyklus}
\end{center}

\begin{itemize}
\item Inputs lesen von der Umgebung
\item Aktionen der aktiven Steps ausführen
\item Guards auswerten und Transitionen nehmen (wenn möglich)
\item Outputs schreiben
\end{itemize}

\Pfoilhead{Slime}

Entwicklungsumgebung für SFC's, bestehend aus den Paketen
\begin{itemize}
\item GUI (graphische Benutzerschnittstelle)
\item Editor
\item Platzierung
\item Parser
\item Checks
\item Simulator
\item Übersetzung nach SMV
\item Model-Checker
\item Codegenerierung
\item Hilfsprogramme (z.B. pretty-printer)
\end{itemize}
